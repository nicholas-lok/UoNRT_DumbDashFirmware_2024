◆ _SK9822_EndFrame()

void _SK9822_EndFrame ( uint16_t  numOfLEDs)

Sends an "End Frame" signal to the LED strip. This is the last step in updating the LED strip if you are using the low-level interface described in the startFrame() documentation.

After this function returns, the clock and data lines will both be outputs that are driving low. This makes it easier to use one clock pin to control multiple LED strips.

◆ _SK9822_StartFrame()

void _SK9822_StartFrame ( )

Mid-Level Protocol Functions

Initialises the I/O lines and sends a "Start Frame" signal to the LED strip.

This is part of the low-level interface provided by this class, which allows you to send LED colors as you are computing them instead of storing them in an array. To use the low-level interface, first call startFrame(), then call sendColor() some number of times, then call endFrame().
